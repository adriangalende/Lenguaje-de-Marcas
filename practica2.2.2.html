¿ XHTML tiene validez?

Claro que tiene validez, y no solo eso, sino que además presenta algunas "mejoras" respecto a HTML

  • Es compatible con todos los navegadores principales !
  • Busca Estandarizar y es más estricto que HTML, así minimiza la cantidad de HTML "Malo" que tienen las páginas web
  • Más liviano que HTML ( punto anterior )

¿ Por qué es más estricto que HTML ?

Vamos a ver algunas de las obligaciones que tiene XHTML respecto a HTML

Elementos

  • Tienen que estar correctamente anidados
  • Se tienen que cerrar siempre !!!
  • su nombre debe estar siempre en minúscula
  • Siempre debe tener el elemento <html>

Atributos

  • Su nombre siempre debe estar en minúscula
  • los valores de los atributos deben estar "citados"

¿ Qué es Strict ?

Contiene todos los elementos y atributos de HTML, pero NO INCLUYE elementos de presentación ni obsoletos. Los Frameset tampoco están permitidos

Elementos presentación

  • < i > - Italic
  • < b > - negrita
  • < u > - subrayado
  • < S > - tachado
  • < sup > - superscript
  • < sub > - subscript ( Para mostrar subtextos, estilo N en base 2 )

Tags obsoletos

  • < applet >
  • < basefont > - pone estilos de fuente
  • < center >
  • < dir >
  • < font >
  • < isindex >
  • < menu >
  • < xmp >
  • < plaintext >
  • < embed >

¿ Qué es transational ?

Es la más común, ya que mezcla Strict con los antiguos tags que conocemos de HTML ( que están ya en desuso por el consorcio que lo regula )

Es un doctype transitorio ( se supone que debería haber dado ya paso a STRICT ).

Con el se da apoyo a navegadores antigüos que soportan mal las hojas de estilos y disponemos de flexibilidad a la hora de escribir el código de nuestra página

Sí, es más permisito pero A COSTA de no entrar de lleno en el estándar y de perder un poco de accesibilidad.

¿ Qué utilizar, Strict o transational ?

Tarde o temprano, los proyectos que tengamos se tendrán que migrar a XHTML 1.1 y la mejor forma de adaptar el código es utilizando STRICT.
